坑一：Spring Cloud 使用 Fegin 调用接口竟然返回了 xml 格式的报文

最近在项目中整合 Spring Cloud 相关组件时，遇到了一个坑，就是使用 Fegin 调用其他服务时，返回的数据结构不是 json，而是莫名其妙变成了 xml 格式，最后查看官方文档才解决。特此记录下。

对方服务的接口返回是用@RestController注解修饰的，直接使用原生 Http 方式调用返回的是 json，证明接口是没问题的。在查看官方文档后才了解是因为引入了 jackson-dataformat-xml这个依赖，这个依赖提供了jackson将实体转化为 xml 的功能。

坑二：Spring Cloud 整合 Zookeeper，因为 Spring Boot 版本的问题导致服务注册不上 Zookeeper

在前面一篇笔记《Zookeeper笔记整理》中介绍过 Eureka、Zookeeper 和 Consul 三种注册中心的对比，公司新项目开展也是打算用 Zookeeper 作注册中心。我在用 Spring Cloud F版整合 Zookeeper 时遇到了很奇怪的问题：

由于Zookeeper的3.5.*版本属于beta版本，线上不建议使用，所以我们使用的是3.4.11版本的 zookeeper，按照官方文档引入依赖：

Spring Cloud 版本：Finchley.RELEASE

Spring Boot 版本：2.1.1RELEASE（当前最新版）

项目启动后发现本项目信息注册不上 Zookeeper，怀疑自己哪里弄错了，查看官方文档也没找到具体解释，最终想到会不会是版本的问题，后来将 Spring Boot 版本换成项目中正在使用的 2.0.3.RELEASE 版本，项目启动后就可以正常注册到 Zookeeper 了，这个是真的是…😒，由于新版本大家都还没用过，所以我只好将问题提到了社区里，等待结果ing…。
